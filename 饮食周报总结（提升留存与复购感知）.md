我们要开发一个能提升用户长期留存的功能：“饮食周报总结”。请全栈实现以下逻辑：
>
> 1. 后端数据聚合与 AI 分析：
>    - 新建 API `GET /api/reports/weekly`：查询当前用户过去 7 天的饮食记录和体重数据。
>    - 计算这 7 天的：平均每日热量摄入、三大营养素（蛋白质、脂肪、碳水）的平均摄入比例、体重的变化趋势（需结合体重表）。
>    - 将这些统计数据作为上下文传给 AI，让 AI 扮演专业营养师，给出一段 150 字左右的本周饮食改善建议。
>
> 2. 前端展示页面：
>    - 创建一个“数据周报”页面。
>    - 用折线图展示这 7 天的“热量摄入趋势”和“蛋白质摄入趋势”。
>    - 用一个卡片展示 AI 的点评和具体的“下周改善建议”（比如：本周蛋白质摄入偏低，下周建议多吃鸡胸肉和鸡蛋）。
>

     1. 新建 API GET /api/reports/weekly

     文件：backend/controllers/reportController.js

     数据聚合逻辑：
     1. 计算 7 天日期范围（今天往前推 6 天）
     2. 调用 MealRecord.getWeeklyRecords(userId, startDate, endDate) — 3 参方法，返回 [{ record_date, calories, protein, fat, carbs }]
     3. 调用 User.findById(userId) — 获取 daily_calorie_target 作为热量目标
     4. 调用 Weight.getTrend(userId, 7) — 返回 [{ weight, recorded_at }]
     5. 统计计算：
       - 7 天平均热量摄入、平均蛋白质/脂肪/碳水（g 和百分比）
       - 体重变化趋势（首日 vs 末日差值 + 方向描述）
     6. 构建 AI prompt（将统计数据格式化为自然语言），调用 callAI() 获取 ~150 字营养师点评
     7. 返回结构化 JSON

     AI Prompt 设计：
     你是一位专业营养师。请根据用户过去7天的饮食数据，给出一段150字左右的周报总结和改进建议。

     数据：
     - 7天平均热量：X kcal（目标：Y kcal）
     - 三大营养素比例：蛋白质 Z%，脂肪 W%，碳水 V%
     - 体重变化：从 A kg 到 B kg（变化 C kg）

     要求：语气亲切专业，指出亮点和不足，给出1-2条具体可执行的改善建议。

     API 响应格式：
     {
       "code": 200,
       "data": {
         "dateRange": { "start": "2026-05-21", "end": "2026-05-27" },
         "dailyData": [

  GET /api/reports/weekly
  Authorization: Bearer <token>

  Response:
  {
    dateRange: { start, end },
    dailyData: [{ date, calories, protein, fat, carbs }],  // 7天，含空白天
    summary: { avgCalories, targetCalories, avgProtein, avgFat, avgCarbs, proteinPct, fatPct, carbsPct },

  功能要点

  - AI 不可用时优雅降级：若 AI API Key 未配置或调用失败，自动使用规则引擎生成点评建议
  - 空白天补全：7 天中无饮食记录的天数补 0 值，折线图连续显示
  - 体重趋势：显示首末日差值 + 方向（上升/下降/持平）
  - 两个 Chart.js 折线图：绿色热量趋势 + 红色蛋白质趋势，销毁-重建模式防止内存泄漏
