  最高效的启动方式

  这是一个 后管前 的三模块项目，启动起来其实很简单：

  # 1. 后端（同时托管用户前端 static 文件）
  cd D:\饮食计划\deity001\backend
  node app.js          # 默认端口 8888，一行启动

  # 2. 管理后台（可选，需要时才启动）
  cd D:\饮食计划\deity001\admin-frontend
  npx vite             # 默认端口 5173

  不需要单独启动 frontend/，后端 Express 的 express.static 已经自动托管前端目录了（见
  app.js:25）。

  条件依赖：
  - MySQL 需要运行（默认 localhost:3306，库名 diet_plan，账号 root/123456）
  - .env 中的 DeepSeek 和阿里云视觉 API Key 用于 AI 功能（不影响核心流程）
  - 数据库初始化脚本在 database/diet_init.sql

  项目 deity001 启动流程

  项目架构（3 个子项目）

  子项目: backend/
  技术栈: Express.js + MySQL
  端口: 8888
  启动方式: node app.js 或 npm run dev
  ────────────────────────────────────────
  子项目: frontend/
  技术栈: 纯 HTML/CSS/JS
  端口: 由后端托管
  启动方式: 无需单独启动（后端托管静态文件）
  ────────────────────────────────────────
  子项目: admin-frontend/
  技术栈: Vue 3 + Vite + Element Plus
  端口: 5173
  启动方式: npx vite 或 npm run dev

  启动顺序

  第一步：确保 MySQL 运行
  - 数据库：localhost:3306，数据库名 diet_plan，用户 root /
  123456
  - 初始化脚本在 database/diet_init.sql

  第二步：启动后端
  cd D:\饮食计划\deity001\backend
  npm start
  后端启动后会：
  1. 加载 .env 环境变量（DeepSeek AI、阿里云视觉模型等）
  2. 连接 MySQL 数据库
  3. 在 8888 端口监听 API 请求
  4. 自动托管 frontend/ 目录的静态文件

  第三步（可选）：启动管理后台
  cd D:\饮食计划\deity001\admin-frontend
  npm run dev
  Vite 开发服务器在 5173 端口启动。
