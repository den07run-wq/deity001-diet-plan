# 饮食计划小程序 - 规划设计文档

## 1. 项目概述

### 1.1 项目背景
随着健康生活理念的普及，越来越多的人开始关注日常饮食管理。本小程序旨在帮助用户轻松制定个性化的饮食计划，记录每日饮食情况，追踪营养摄入，并提供科学的饮食建议。

### 1.2 项目目标
- 为用户提供便捷的饮食计划制定工具
- 帮助用户管理每日饮食与营养摄入
- 提供丰富的食材库和食谱推荐
- 支持多用户、多饮食计划管理

### 1.3 目标用户
- 健身爱好者（需要控制蛋白质/碳水摄入）
- 减脂/增肌人群
- 注重健康饮食的普通用户
- 需要饮食管理的特殊人群（如糖尿病患者等）

## 2. 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Node.js + Express.js | 与现有项目保持一致，复用技术栈 |
| 数据库 | MySQL | 与现有项目保持一致 |
| 前端 | HTML + CSS + JavaScript | 原生技术栈，无框架依赖 |
| 用户认证 | JWT (JSON Web Token) | 无状态认证方案 |
| 密码加密 | bcrypt | 安全的密码哈希方案 |
| 构建工具 | npm | Node.js 标准包管理 |

## 4. 功能模块设计

### 4.1 用户管理模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户注册 | 用户名+密码+基本信息注册 | P0 |
| 用户登录 | JWT Token 登录认证 | P0 |
| 个人信息管理 | 修改身高、体重、年龄、性别、健康目标等 | P1 |
| 修改密码 | 修改登录密码 | P1 |

### 4.2 食材/菜品库模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 食材分类浏览 | 按类别（主食、肉类、蔬菜、水果等）浏览 | P0 |
| 食材搜索 | 按名称搜索食材 | P0 |
| 食材详情 | 查看食材的营养成分（热量、蛋白质、脂肪、碳水等） | P0 |
| 自定义食材 | 用户自行添加食材数据 | P1 |
| 食谱推荐 | 根据用户目标推荐食谱组合 | P2 |

### 4.3 饮食计划模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建饮食计划 | 设置计划名称、目标（减脂/增肌/保持）、周期 | P0 |
| 编辑饮食计划 | 修改计划内容，调整食材和分量 | P0 |
| 删除饮食计划 | 删除不需要的计划 | P1 |
| 按天/周查看计划 | 日历式查看每日饮食安排 | P0 |
| 计划模板 | 预设常见饮食计划模板 | P1 |
| 每日营养目标 | 自动计算每日热量和三大营养素目标 | P0 |

### 4.4 饮食记录模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 记录每餐饮食 | 记录早餐/午餐/晚餐/加餐 | P0 |
| 打卡功能 | 标记今天是否按计划执行 | P1 |
| 饮食日记 | 按日查看历史饮食记录 | P1 |
| 食物拍照记录 | 上传饮食照片（可选） | P2 |

### 4.5 营养分析模块
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 每日营养概览 | 当天热量、蛋白质、脂肪、碳水摄入统计 | P0 |
| 周度/月度报告 | 营养摄入趋势图表 | P1 |
| 目标对比 | 实际摄入 vs 目标摄入对比分析 | P1 |
| 饮食建议 | 根据数据生成改进建议 | P2 |

## 5. 数据库设计

### 5.1 用户表 (users)
| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | 加密后的密码 |
| nickname | VARCHAR(50) | | 昵称 |
| gender | ENUM('男', '女') | | 性别 |
| age | INT | | 年龄 |
| height | DECIMAL(5,2) | | 身高(cm) |
| weight | DECIMAL(5,2) | | 体重(kg) |
| goal | ENUM('减脂', '增肌', '保持健康', '其他') | DEFAULT '保持健康' | 健康目标 |
| daily_calorie_target | INT | | 每日目标热量(kcal) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 5.2 食材表 (food_items)
| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 食材ID |
| name | VARCHAR(100) | NOT NULL | 食材名称 |
| category | ENUM('主食', '肉类', '蛋奶', '豆类', '蔬菜', '水果', '零食', '饮品', '调味料', '其他') | NOT NULL | 食材分类 |
| calories_per_100g | DECIMAL(8,2) | NOT NULL | 每100g热量(kcal) |
| protein_per_100g | DECIMAL(6,2) | | 每100g蛋白质(g) |
| fat_per_100g | DECIMAL(6,2) | | 每100g脂肪(g) |
| carbs_per_100g | DECIMAL(6,2) | | 每100g碳水化合物(g) |
| fiber_per_100g | DECIMAL(6,2) | | 每100g膳食纤维(g) |
| unit | VARCHAR(20) | DEFAULT '克' | 计量单位 |
| image_url | VARCHAR(500) | | 食材图片URL |
| is_custom | TINYINT(1) | DEFAULT 0 | 是否用户自定义(0=系统预设,1=用户添加) |
| created_by | INT | FOREIGN KEY REFERENCES users(id) | 创建者(自定义食材时使用) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 5.3 饮食计划表 (diet_plans)
| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 计划ID |
| user_id | INT | FOREIGN KEY REFERENCES users(id), NOT NULL | 所属用户 |
| name | VARCHAR(100) | NOT NULL | 计划名称 |
| goal | ENUM('减脂', '增肌', '保持健康', '其他') | NOT NULL | 计划目标 |
| start_date | DATE | NOT NULL | 开始日期 |
| end_date | DATE | NOT NULL | 结束日期 |
| daily_calories | INT | NOT NULL | 每日目标热量 |
| daily_protein | DECIMAL(6,2) | | 每日目标蛋白质(g) |
| daily_fat | DECIMAL(6,2) | | 每日目标脂肪(g) |
| daily_carbs | DECIMAL(6,2) | | 每日目标碳水(g) |
| is_template | TINYINT(1) | DEFAULT 0 | 是否模板计划 |
| status | ENUM('进行中', '已完成', '已取消') | DEFAULT '进行中' | 计划状态 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 5.4 计划食材关联表 (plan_foods)
| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 关联ID |
| plan_id | INT | FOREIGN KEY REFERENCES diet_plans(id), NOT NULL | 计划ID |
| food_id | INT | FOREIGN KEY REFERENCES food_items(id), NOT NULL | 食材ID |
| day_of_week | ENUM('周一','周二','周三','周四','周五','周六','周日') | NOT NULL | 星期几 |
| meal_type | ENUM('早餐', '午餐', '晚餐', '加餐') | NOT NULL | 餐次类型 |
| quantity | DECIMAL(8,2) | NOT NULL | 数量(克) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 5.5 饮食记录表 (meal_records)
| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | FOREIGN KEY REFERENCES users(id), NOT NULL | 用户ID |
| plan_id | INT | FOREIGN KEY REFERENCES diet_plans(id) | 关联计划(可为空) |
| food_id | INT | FOREIGN KEY REFERENCES food_items(id), NOT NULL | 食材ID |
| record_date | DATE | NOT NULL | 记录日期 |
| meal_type | ENUM('早餐', '午餐', '晚餐', '加餐') | NOT NULL | 餐次类型 |
| quantity | DECIMAL(8,2) | NOT NULL | 实际食用量(克) |
| note | VARCHAR(500) | | 备注 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 6. API 接口设计

### 6.1 用户相关接口

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/user/register | 用户注册 | 否 |
| POST | /api/user/login | 用户登录，返回JWT Token | 否 |
| GET | /api/user/profile | 获取当前用户信息 | 是 |
| PUT | /api/user/profile | 更新用户信息（身高/体重/目标等） | 是 |
| PUT | /api/user/password | 修改密码 | 是 |

### 6.2 食材相关接口

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/food | 获取食材列表（支持分类筛选和搜索） | 否 |
| GET | /api/food/:id | 获取食材详情 | 否 |
| GET | /api/food/categories | 获取食材分类列表 | 否 |
| POST | /api/food/custom | 添加自定义食材 | 是 |
| DELETE | /api/food/custom/:id | 删除自定义食材 | 是 |

### 6.3 饮食计划接口

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/plan | 获取当前用户所有计划 | 是 |
| POST | /api/plan | 创建新饮食计划 | 是 |
| GET | /api/plan/:id | 获取计划详情（含每日食材安排） | 是 |
| PUT | /api/plan/:id | 更新饮食计划 | 是 |
| DELETE | /api/plan/:id | 删除饮食计划 | 是 |
| GET | /api/plan/:id/day/:day | 获取计划中某天的饮食安排 | 是 |
| POST | /api/plan/:id/food | 向计划中添加某天的食材 | 是 |
| PUT | /api/plan/food/:planFoodId | 更新计划中某食材的分量 | 是 |
| DELETE | /api/plan/food/:planFoodId | 从计划中移除某食材 | 是 |
| GET | /api/plan/templates | 获取系统预设计划模板 | 否 |

### 6.4 饮食记录接口

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/record?date=YYYY-MM-DD | 获取指定日期的饮食记录 | 是 |
| POST | /api/record | 添加饮食记录 | 是 |
| PUT | /api/record/:id | 更新饮食记录 | 是 |
| DELETE | /api/record/:id | 删除饮食记录 | 是 |
| GET | /api/record/weekly?start=YYYY-MM-DD | 获取一周饮食记录 | 是 |

### 6.5 营养分析接口

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/nutrition/daily?date=YYYY-MM-DD | 获取某天营养摄入统计 | 是 |
| GET | /api/nutrition/weekly?start=YYYY-MM-DD | 获取一周营养统计 | 是 |
| GET | /api/nutrition/monthly?year=YYYY&month=MM | 获取月度营养趋势 | 是 |

### 6.6 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

| code | 含义 |
|------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 9. 核心业务逻辑

### 9.1 每日营养目标自动计算

系统根据用户的身体数据和目标自动推荐每日营养素摄入量：

**基础代谢率 (BMR) 计算**（Mifflin-St Jeor 公式）：
- 男性：`BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 5`
- 女性：`BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161`

**每日总热量目标 (TDEE)**：
| 目标 | 计算公式 |
|------|----------|
| 减脂 | TDEE = BMR × 1.2 - 300 |
| 增肌 | TDEE = BMR × 1.2 + 300 |
| 保持健康 | TDEE = BMR × 1.2 |

**三大营养素推荐配比**：
| 目标 | 蛋白质 | 脂肪 | 碳水 |
|------|--------|------|------|
| 减脂 | 35% | 25% | 40% |
| 增肌 | 30% | 20% | 50% |
| 保持健康 | 25% | 25% | 50% |

### 9.2 营养摄入统计计算

用户记录饮食后，系统自动计算：

```
实际摄入热量 = Σ(食材热量_per_100g × 食用量 / 100)
实际摄入蛋白质 = Σ(食材蛋白质_per_100g × 食用量 / 100)
实际摄入脂肪 = Σ(食材脂肪_per_100g × 食用量 / 100)
实际摄入碳水 = Σ(食材碳水_per_100g × 食用量 / 100)
```

### 9.3 饮食建议生成规则

| 条件 | 建议内容 |
|------|----------|
| 连续3天热量超标 > 20% | 建议减少高热量食物摄入 |
| 连续3天热量不足 > 30% | 建议增加餐食分量，防止营养不良 |
| 蛋白质连续低于目标 70% | 建议增加肉、蛋、奶、豆类摄入 |
| 脂肪连续超过目标 30% | 建议减少油炸食品和高脂肉类 |
| 碳水严重不足 | 建议增加主食摄入量 |

## 11. 安全设计

| 措施 | 说明 |
|------|------|
| 密码加密 | 使用 bcrypt 加盐哈希，不存储明文密码 |
| JWT 认证 | 无状态 Token，过期时间默认 7 天 |
| SQL 注入防护 | 使用参数化查询（mysql2 的 prepared statements） |
| XSS 防护 | 前端输出时转义用户输入内容 |
| 权限校验 | 用户只能操作自己的计划和记录 |
